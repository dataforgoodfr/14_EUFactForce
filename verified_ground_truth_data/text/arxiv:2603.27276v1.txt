When dealing with complex hierarchical models, the MCMC approach can be computationally demanding in workflows that require repeated model fitting or when working with models of large dimensions with limited hardware resources. The Integrated Nested Laplace Approximations (INLA) is a deterministic alternative for models with non-Gaussian data that belong to the class of latent Gaussian models (LGMs), yielding accurate approximations to posterior marginals in many applied settings. The INLA method was implemented in C as a standalone program, inla, that is widely used in R through the INLA package. This paper introduces PyINLA, a dedicated Python package that provides a Pythonic interface directly to the inla program. Therefore, PyINLA enables specifying LGMs, running INLA-based inference, and accessing posterior summaries directly from Python while leveraging the established INLA implementation. In many application areas, including spatial epidemiology Lawson2018, environmental and ecological modeling Cressie2011, and econometrics Koop2003, the practical value of Bayesian modeling depends on computational tools that are fast, reliable, and easy to integrate into end-to-end analysis pipelines. In contemporary software ecosystems, general-purpose Bayesian inference is most often performed using Markov chain Monte Carlo (MCMC) or variational inference (VI). These approaches underpin mature probabilistic programming frameworks, but they carry well-known trade-offs: MCMC is computationally intensive for models with high-dimensional latent structure and requires careful convergence assessment, while mean-field VI introduces approximation bias and tends to underestimate posterior uncertainty Brooks2011, Blei2017. These trade-offs become acute in practice for hierarchical models with large latent fields, iterative workflows involving repeated model fitting (sensitivity analysis, model selection, cross-validation), and production settings that demand deterministic, reproducible outputs. The Integrated Nested Laplace Approximations (INLA) provides a deterministic alternative for an important and practically broad class of models: the latent Gaussian models (LGMs) Rue2009. The Laplace approximation exploits the conditional structure in the prior distribution of the latent field twice. One while searching for the posterior mode of the hyperparameters and then several nested Laplace approximations combined with numerical integration over the hyperparameters. These nested Laplace approximations in the INLA algorithm can be replaced by an implicit low-rank correction, as proposed in van2024vbc, which is shown to be faster than the original INLA and as accurate, van2023new. When the latent precision matrix is sparse, as it is for Gaussian Markov random fields rue2005gaussian and a wide class of structured random effects, INLA computes accurate approximations to posterior marginal distributions efficiently, even when the latent dimension is large. These properties have established INLA as a competitive method for spatial and spatiotemporal hierarchical models across applied statistics Blangiardo2013, Martins2013,competitionSpatial. The reference implementation of INLA is the R-INLA project (https://www.r-inla.org), which has matured into a comprehensive and widely used framework in the R ecosystem lindgren2015bayesian, bivand2015spatial, van2021new. Several R packages extend its scope to broader spatial models, non-separable spatio-temporal structures, non-linear predictors, and joint longitudinal-survival models rSPDE, krainski2025inlaspacetime, bachl2019inlabru, lindgren2024inlabru, rustand2024joint, with applications spanning biostatistics, geosciences, epidemiology, and environmental modeling alvares2024bayesian, fioravanti2023interpolating, myer2019spatiotemporal, seaton2024spatio. Practitioners in Python, now the dominant language in data science and machine learning, which need fast Bayesian inference for hierarchical models with spatial, temporal, or grouped structure have largely been limited to MCMC-based tools such as PyMC or Stan, which can be computationally demanding for models with large latent fields and requires careful convergence assessment. INLA provides a deterministic alternative for these models, but has until now lacked a native Python interface. PyINLA fills this gap: it is a self-contained Python package, with no R installation required, that brings INLA-based inference directly form the standalone inla C program into standard Python workflows built around pandas, NumPy, and SciPy. This paper introduces PyINLA, a native Python package that brings INLA-based inference directly into the Python ecosystem. PyINLA exposes the compiled INLA computational engine through a Pythonic API, enabling users to specify LGMs, run inference, and access full posterior summaries using standard Python data structures (pandas DataFrames, NumPy arrays, SciPy sparse matrices). The package eliminates the need for an R installation or cross-language interoperability layers, making INLA accessible to the large and growing community of practitioners who build their analysis and deployment pipelines in Python. The package currently supports widely used INLA likelihood families and latent model structures, including group-level random effects, temporal (random walks, AR processes), and spatial (BYM, SPDE), using principled prior specification via penalized complexity priors simpson2017penalising. Background: Bayesian computation and related softwaresec:background Applied Bayesian workflows involve iterative model development, repeated fitting across related datasets, and routine model checking. In these settings, the choice of inference method is driven not only by statistical objectives but also by runtime, diagnostic burden, and integration into data-processing and deployment pipelines. Within Python, general-purpose Bayesian inference is commonly carried out using MCMC or VI, implemented in frameworks such as PyMC, Stan (via CmdStanPy), TensorFlow Probability, and Pyro Salvatier2016, Carpenter2017, Dillon2017, Bingham2019. Modern MCMC methods such as Hamiltonian Monte Carlo and the No-U-Turn Sampler Hoffman2014 provide asymptotically exact inference but are expensive for hierarchical models with large latent structure and require substantial effort for tuning and convergence assessment. Variational methods replace sampling with optimization and reduce runtime, but they introduce bias and understate posterior uncertainty when posterior dependence is strong Rue2009,Zhang2019. These trade-offs motivate deterministic alternatives for model classes where such methods are applicable. INLA provides a deterministic alternative for latent Gaussian models by replacing sampling with analytical approximations, as outlined in Section~sec:introduction and detailed in Section~sec:inla Rue2009, Rue2017. Empirical comparisons consistently demonstrate that INLA achieves comparable accuracy to MCMC while being one to two orders of magnitude faster across various applications, including disease mapping, clinical trials, and financial modeling de2015comparing, chen2024comparison, darkwah2022bayesian, nacinben2024multivariate. PyINLA provides a Python-native interface to the compiled INLA engine, making deterministic Bayesian inference for LGMs directly accessible within Python workflows. For the large class of models within its scope, this design yields concrete practical advantages: itemize Efficiency for structured latent fields. The INLA methodologysec:inla This section summarizes the latent Gaussian model framework and the INLA approximation strategy at the level required to understand what PyINLA can represent and what it returns. Detailed methodological treatments are available in Rue2009, abdul2023approximate, van2024vbc. Before the formal definition, it helps to build up the model structure from data. Suppose we observe data $y_i$ that could be continuous measurements (e.g., temperature), counts (e.g., number of cases), or binary outcomes (e.g., success/failure). We want to model the expected response $_i = E[y_i]$ as a function of covariates and random effects. Since $_i$ is often restricted (e.g., positive for counts, in $(0,1)$ for probabilities), a link function $g()$ maps it to the real line. The transformed response $_i = g(_i)$ is called the linear predictor, modeled as: equationeq:general-pattern-preview _i = g(_i)_{linear predictor} = _0 + _1 x_{1i + }_{fixed effects} + u^{(1)_{k_1(i)} + u^{(2)}_{k_2(i)} + }_{random effects}, y_i p( _i, {)}_{likelihood}. equation Here $$ are fixed regression coefficients and $u^{(k)}$ are structured random effects capturing variation due to spatial proximity, temporal trends, or group membership. Together, they form the latent field $x = (_0, , u_1, , u_K)^$, which is assigned a joint Gaussian prior as detailed below. Latent Gaussian models A latent Gaussian model (LGM) can be defined following a hierarchical structure, which includes 1) a family (or likelihood) assumed for the data, 2) Gaussian priors for the parameters in the linear predictor (the latent field), and 3) a hyperparameter vector. Let $y=(y_1,,y_n)^$ denote observed data, $x^N$ the latent field as introduced above, and $^p$ hyperparameters. A simplified view follows; see Rue2017 for full details. Observation model. The data are conditionally independent given $x$ and $$: equation y_i x, {ind}{} p(y_i _i,), _i = a_i^ x, equation where $a_i^$ is typically a sparse row of the design matrix mapping the latent field to the linear predictor. A wide range of density functions can be used as a likelihood $p()$. The only technical requirement is that it needs to have first and second order derivatives with respect to the latent field. The INLA engine supports a large catalogue of likelihood families as well as user-defined likelihoods; PyINLA currently exposes the most commonly used subset (see Section~sec:families). Latent field prior. Conditional on $$, equation x N\!( (),\ Q()^{-1}), equation where $Q()$ is a precision matrix (the inverse of the covariance matrix). In most structured models, $Q()$ is sparse, encoding conditional independence relationships that are central to computational efficiency rue2005gaussian. Hyperparameter prior. The hyperparameters are the extra likelihood parameters (not in the linear predictor) such as variance, dispersion, zero-inflation and the parameters on $Q()$. They are assigned a prior distribution $p()$, often including penalized complexity priors for variance and correlation parameters simpson2017penalising. PC priors are specified through intuitive probability statements rather than abstract distributional parameters. For example, $P( > 1) = 0.01$ encodes the belief that a standard deviation exceeding 1 is unlikely, shrinking toward the simpler base model with no random effect. Fixed effects are specified via fixed and random effects via random (see Section~sec:pyinla). Inference targets and outputs INLA targets posterior marginal distributions for each element of the latent field and hyperparameters: equation p(x_j y) = p(x_j , y)\,p( y)\, d, p(_k y) = p( y)\,d_{-k}. equation These marginal distributions are non-Gaussian, and the Laplace method exploits the Gaussian prior on $x$ to approximate them accurately and efficiently. Once obtained, posterior summaries such as means, standard deviations, and credible intervals are computed directly from these marginals, and returned as pandas DataFrames (see Section~sec:pyinla). INLA focuses on marginals rather than representing the full joint posterior explicitly; this design is a key contributor to its computational efficiency. When the full joint posterior is needed, Monte Carlo samples can be drawn efficiently after the model fit joint2021, as described in Section~sec:pyinla. Nested Laplace approximation and numerical integration INLA proceeds by combining (i) Laplace approximations for high-dimensional Gaussian latent structure with (ii) numerical integration over the hyperparameter space Rue2009, Rue2017. At a high level: enumerate Construct a Gaussian approximation to $p(x ,y)$ to compute a Laplace approximation to $p()$ and use it to search for the posterior mode of $$. Approximate the conditional marginals $p(x_j ,y)$, for a set of support points $$ around its mode. Integrate over $$ using the set of support points with weights proportional to $p()$ to obtain $p(x_j y)$ and $p(_k y)$. enumerate Note: if the likelihood is Gaussian, no approximation is required in steps 1 and 2. Computational considerations and limitations The first step of INLA is an optimization to locate the mode of the hyperparameter, and uses an adaptive gradient estimation technique fattah2022smart. The inla program exploits sparsity in the latent precision matrix via Cholesky factorization and selective inversion rue2005gaussian. When the precision matrix is sparse, these operations are fast, but they become a bottleneck if the precision matrix is dense. Prior work explored solutions for specific cases, including dense matrices abdul2025inla+ and large-scale spatiotemporal models gaedke2024integrated. The sTiles library fattah2025stiles, fattah2025gpu is being developed to provide efficient factorization for sparse structured matrices, with GPU acceleration planned for future releases. The inla source code is available at https://github.com/hrue/r-inla. The interface descriptionsec:pyinla This section describes the design, architecture, and usage of PyINLA, a Python-native interface to the compiled inla program that enables model specification, execution, and result handling entirely within standard Python workflows. Installation and availability PyINLA can be installed from PyPI: CodeChunk CodeInput pip install pyinla CodeInput CodeChunk Optional extras are available for workflows that require additional functionality, for example mesh construction utilities for SPDE-based spatial models: CodeChunk CodeInput pip install pyinla[fmesher] CodeInput CodeChunk The [fmesher] extra provides Python wrappers around the fmesher library lindgren2023fmesher for mesh generation and SPDE preprocessing. We recommend installation in an isolated environment (e.g., conda). PyINLA is a standard Python package with no R dependency. The heavy lifting is done by the inla program, a compiled C binary that implements the core numerical routines, sparse matrix factorizations, Laplace approximations, and numerical integration over hyperparameters. This is the same engine that has powered R-INLA for over 15 years and has been validated across thousands of published analyses. PyINLA manages this binary transparently: it is automatically downloaded on first use if not already present, so no manual configuration is required. The package runs directly in cloud notebook environments such as Google Colab, making it suitable for teaching, reproducible demonstrations, and collaborative research without local installation. Full documentation, tutorials, and interactive examples are available at https://pyinla.org. To support reproducible research, the repository includes replication materials for all results reported in this manuscript. This paper describes PyINLA version 0.2.0. Package architecture PyINLA is organized into five layers: enumerate User-facing API: A high-level interface centered on the pyinla() function, which accepts model specification, data, and control options and returns a structured result object. Model specification layer: Translation of Python-level model definitions into the internal representation required by the INLA engine, including validation and defaults. Data preparation layer: Conversion of common Python objects, pandas DataFrames, NumPy arrays, and SciPy sparse matrices, into formats consumed by the engine. Execution layer: Orchestration of INLA runs, including management of working directories, configuration generation, invocation of the compiled engine, and error reporting. Result collection layer: Parsing of engine outputs and construction of Python result objects that expose posterior marginals, summaries, and diagnostics. enumerate Basic usage A minimal example fitting a Gaussian linear model illustrates the primary workflow. From equation to dictionary: a progressive construction Step 1: A linear model. Consider the simplest case: a Gaussian linear model with one covariate, equationeq:step1 y_i = _0 + _1 x_i + _i, _i N(0, ^2). equation In PyINLA, this model is specified as: CodeChunk CodeInput model = {"response": "y", "fixed": ["1", "x"]} result = pyinla(model=model, family="gaussian", data=data) CodeInput CodeChunk The dictionary has two keys. The "response" key names the column in data that contains the observed values $y_i$. The "fixed" key lists the terms in the linear predictor: "1" denotes the intercept $_0$, and "x" denotes the covariate column whose coefficient is $_1$. The argument family="gaussian" specifies Gaussian observations with an identity link, meaning the linear predictor equals the conditional mean directly: $_i = E[y_i] = _0 + _1 x_i$. Step 2: Changing the likelihood and link function. Discussion and conclusion sec:conclusion This paper introduced PyINLA, a native Python package that brings INLA-based Bayesian inference to the Python ecosystem. By providing direct access to the compiled INLA computational engine through a Pythonic API, PyINLA eliminates the friction of cross-language workflows while preserving the computational efficiency and statistical accuracy that have made INLA a standard tool for latent Gaussian models. The examples in Section~sec:examples demonstrated PyINLA across a range of model structures, from exchangeable random effects for sports prediction to spatially structured effects for disease mapping, continuous spatial fields for geostatistical interpolation, and temporal components for time series forecasting. Across these applications, posterior estimates closely matched MCMC benchmarks while achieving computational speedups of 100--200$$, with deterministic outputs that simplify testing and deployment. Several limitations should be acknowledged. PyINLA is restricted to latent Gaussian models; applications requiring non-Gaussian latent structure or likelihoods outside INLA's supported families require general-purpose tools. Applications requiring direct access to the full joint posterior may also benefit from MCMC methods, although PyINLA provides posterior sampling for derived quantities. Development continues along several axes, including extended model support, GPU acceleration via the sTiles library fattah2025stiles, fattah2025gpu, deeper integration with Python machine learning workflows, and expanded documentation at https://pyinla.org. Future releases will also allow users to define custom latent models and likelihood functions, extending PyINLA beyond the built-in components. PyINLA makes INLA-based Bayesian inference a native part of the Python scientific computing ecosystem. By providing direct access to fast, deterministic inference for latent Gaussian models, PyINLA opens the door to tighter integration with modern machine learning and AI workflows, where principled uncertainty quantification is increasingly valued. We anticipate that native Python availability will expand the reach of INLA methodology to new application domains and larger-scale problems. plainnat mybib Marginal posterior utility functionsapp:marginals After model fitting, PyINLA returns marginal posterior distributions $p(_i y)$ for each model component (fixed effects, hyperparameters, random effects, and linear predictor values).
